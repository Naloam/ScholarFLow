# MVP Workflow（低代码）

目标：主题输入 → 检索 10 篇论文 → 文献综述摘要 → 模板草稿 → 简版 Review。

## 节点设计

1. 输入节点
- 用户输入：研究主题、研究范围、语言、输出长度
- HITL：用户确认关键词与范围

2. 检索节点
- 输入：关键词 + 可选限制（年份、领域）
- 输出：10 篇论文元数据（题目、作者、年份、摘要、DOI/URL）
- 失败回退：降低筛选阈值或扩大年份范围
- HITL：用户确认保留/剔除论文

3. 综述摘要节点
- 输入：已选论文元数据与摘要
- 输出：结构化综述摘要（主题脉络、分支、代表性工作）
- 失败回退：缩短篇幅、减少维度
- HITL：用户确认结构与关键点

4. 草稿生成节点
- 输入：综述摘要 + 用户指定模板类型
- 输出：论文草稿（引言/相关工作/方法/结论占位）
- 失败回退：先生成引言与相关工作
- HITL：逐段确认与编辑

5. 简版 Review 节点
- 输入：草稿
- 输出：7 维评分简版 + 三条优先改进建议
- 失败回退：仅输出问题列表
- HITL：用户选择是否回到草稿节点修改

## 质量与合规

- Claim 必须与 Evidence 绑定，缺证据标记为 [NEEDS_EVIDENCE]
- 引用格式统一，输出引用列表
- 明确提示为写作辅助，用户需审核
